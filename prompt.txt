Контекст:
Проект NewVision (ранее Curtain Reader) — Telegram Mini App для чтения английских технических статей с переводом.
GitHub: DrainRanger92/newvision, ветка main.

Текущее состояние:
- M1 ✅ Skeleton (FastAPI + aiogram 3 bot + React TS frontend)
- M2 ✅ Parser (URL → блоки, SQLite кэш)
- M3 ✅ Translation (DeepSeek API EN→RU, lazy cache)
- M4 ⏳ Bot integration (текущая задача)
- M5 ⏳ Curtain UX (touch swipe) — НЕ ТРОГАТЬ

Архитектура:
- backend/: FastAPI приложение (Python/aiogram 3), frontend/: React 18 + Vite + TS

Техстек:
- Backend: Python 3.11+, FastAPI, aiogram 3.x, SQLite (aiosqlite), DeepSeek API
- Frontend: React 18, TypeScript, Vite, @twa-dev/sdk

Порядок:
1. Прочитай весь существующий код
2. Спроектируй изменения

Проблема:
M4 — интеграция бота с Mini App. Сейчас бот отвечает только на /start.
Нужен сквозной сценарий: пользователь кидает URL → бот парсит через POST /api/parse → WebAppInfo кнопка → Mini App открывает и рендерит статью.
