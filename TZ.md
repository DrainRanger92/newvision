# NewVision — Техническое задание (MVP)

## Продукт
Telegram Mini App для чтения английских технических статей с inline-переводом.
Жест «шторка» — свайп вверх по абзацу reveals перевод.

## Пользователь
Изучает технические материалы (программирование, AI/ML) на английском.
Английский не идеальный — нужен быстрый доступ к переводу без потери фокуса.

---

## Пользовательский сценарий

1. Пользователь читает статью (Real Python, MDN, блоги)
2. Кидает URL в чат с ботом
3. Бот парсит статью, готовит Mini App
4. Пользователь нажимает кнопку → открывается статья прямо в Telegram
5. Читает оригинал → непонятный абзац → свайп вверх → видит перевод
6. Отпускает → возвращается к оригиналу
7. Кодовые блоки — не переводятся, отображаются как есть

---

## Функциональные требования

### Бот
- Принимает URL от пользователя
- Возвращает кнопку для открытия статьи в Mini App
- Повторный запрос той же статьи — мгновенно (кэш)

### Чтение статьи
- Статья открывается внутри Telegram (Mini App)
- Отображается в чистом reader-mode (без навигации сайта, рекламы, боковых панелей)
- Заголовок статьи + текст + кодовые блоки + картинки
- Кодовые блоки визуально отличаются от текста (подсветка синтаксиса или моноширинный шрифт + фон)

### Шторка (core feature)
- Каждый абзац текста — интерактивный блок с «шторкой»
- Свайп вверх по абзацу → оригинальный текст уходит вверх, появляется перевод
- Threshold: ~30% высоты блока — шторка «залипает» в открытом состоянии
- Свайп вниз (или повторный жест) → шторка закрывается, возвращается оригинал
- Анимация плавная, без рывков
- Кодовые блоки — без шторки, не реагируют на жест

### Перевод
- Только текст статьи (абзацы, заголовки, списки, цитаты)
- Код внутри абзацев (инлайн-код) — сохраняется как есть, не переводится
- Перевод загружается лениво: первый свайп → запрос перевода → показ
- После загрузки — кэшируется, повторный свайп мгновенный
- Предзагрузка: при скролле — заранее переводит 2-3 следующих блока

### Тема
- Автоматически подхватывает тему Telegram (светлая / тёмная)
- Шрифт и размер — читаемый по умолчанию

---

## Нефункциональные требования

- Первое открытие статьи: < 3 секунды (парсинг + рендер)
- Перевод одного абзаца: < 1.5 секунды
- Перевод из кэша: мгновенно
- Анимация шторки: плавная (60fps)
- Работает на iOS 14+ и Android 8+
- Статья, которую уже открыли, доступна повторно без перепарсинга

## Deployment Contract (Cloud Run)

1. **No message loss on restart** — при перезапуске контейнера (SIGTERM, deploy, scale-down) 
   ни одно сообщение пользователя не должно быть потеряно. Bot корректно завершает обработку 
   текущих update'ов, непроцессированные update'ы доставляются новому инстансу Telegram'ом.

2. **Stateless application** — приложение не хранит состояние в локальной fs контейнера. 
   Все данные (БД, кэш) — во внешних хранилищах (GCS volume, external DB). 
   Контейнер может быть уничтожен и пересоздан в любой момент без потери данных.

3. **Resilience to duplicates** — повторная доставка update'а от Telegram (at-least-once) 
   не вызывает ошибок. Повторный ответ пользователю допустим для MVP.
   **Отложено:** exactly-once semantics при появлении жалоб или денежных транзакций.

## Не входит в MVP

- Выбор языка перевода (только EN → RU)
- «Перевести всё» одним кликом
- Саммари статьи
- Закладки / история прочтения
- PWA standalone (вне Telegram)
- Перевод таблиц
- Desktop-режим (hover вместо swipe)
- Оффлайн-режим

---

## Ожидаемый результат

Рабочий Telegram бот, который:
1. Принимает ссылку на статью
2. Отдаёт кнопку для открытия
3. Показывает статью с рабочей «шторкой»-переводом
4. Код не ломает и не переводит
5. Плавно работает на телефоне
